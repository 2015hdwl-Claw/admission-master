## Part 1：高職升學制度深度研究

### 1.1 六大入學管道

| # | 管道 | 說明 | 門檻 | 適合誰 |
|---|------|------|------|--------|
| 1 | **甄選入學** | 統測成績 + 指定項目甄試（面試/實作/備審） | 統測成績 | 綜合能力強者 |
| 2 | **登記分發入學** | 純統測成績分發 | 統測成績 | 筆試型學生 |
| 3 | **技優甄審** | 技藝優良不需統測，用證照/競賽成績申請 | 乙級證照或技藝競賽獲獎 | 有證照/競賽成績者 |
| 4 | **技優保送** | 國際技能競賽/全國技藝競賽優勝直接保送 | 全國前三名 | 頂尖技能選手 |
| 5 | **繁星推薦** | 校內排名 + 統測門檻 | 校排名前 30% | 成績穩定者 |
| 6 | **跨考學測** | 高職生報考一般高中學測，申請一般大學 | 學測成績 | 想轉跑道的學生 |

**關鍵差異（vs 一般高中）**：
- 一般高中只有學測 + 分科測驗 + 繁星 + 個人申請
- 高職有技優管道（證照和競賽成績可以直接換門票）
- 高職統測按類群分組，不是全科統考

**核心發現**：工科（機械/電機/化工/土木）證照體系完整，升學路最清晰。文科/商科/外語/設計缺少證照支撐，學生不知道自己還有什麼路可走。

### 1.3 甄選入學計分公式

```
總分 = [統測成績 × A% + 指定項目甄試 × B%] × (1 + 優待加分%)
```

- **A% 和 B%** 由各校系自訂，A 通常在 40%-60%，B 在 40%-60%
- **統測成績**：滿分 700 分（每科 100 分 × 7，含共同科目）
- **指定項目甄試**：面試、實作、備審資料審查
- **優待加分%**：疊加制，可累計

### 1.4 優待加分體系

| 加分項目 | 加分比例 | 備註 |
|----------|----------|------|
| 丙級技術士證照 | +5% | 基礎級 |
| 乙級技術士證照 | +15% | 進階級，也是技優甄審門票 |
| 甲級技術士證照 | +25% | 最高級 |
| 技藝競賽獲獎 | 依名次 | 校/縣/全國 |
| 全國技能競賽獲獎 | 依名次 | 可保送 |
| 其他特定身分 | 依規定 | 原住民/身心障礙/弱勢 |

**關鍵洞察**：乙級證照 = +15% 加分 + 技優甄審門票。這是一張「雙重價值」的門票，但很多文科/商科生不知道。

| 多元表現 | C-3 | 擔任幹部 | 每年 2 件 | 領導力 |
| 多元表現 | C-4 | 競賽成果 | 每年 2 件 | **高價值** |

### 1.6 116 學年度（2027 入學）重大改革

#### 變革一：自選考科
- 現制：統測 5 科全考（國+英+數+專一+專二）
- 116 起：自選 2-5 科應考
- 影響：學生可以放棄弱科，專攻強科

#### 變革二：分配計分
- 各校系自訂採計科目與加權
- 不同校系可能只看 3 科或 4 科
- 選考組合直接影響可申請的校系範圍

#### 變革三：策略需求爆發
- 學生需要知道：目標校系採計哪些科目？
- 哪些科目組合可以覆蓋最多目標校系？
- 放棄某科會失去哪些校系的申請資格？
- 這是一個**全新的決策需求**，目前沒有任何平台提供

#### 產品機會：116 選考策略引擎

```
輸入：學生的目標校系列表
輸出：最優考科組合 + 每個組合可覆蓋的校系 + 風險評估
```

這個引擎需要：
1. 各校系 116 招生簡章的採計科目資料
2. 學生的各科能力自評
3. 組合最佳化演算法

## Part 2：供給端研究

### 2.1 各類群招生規模

| 類群 | 年招生人數（估） | 跨考率 | 備註 |
|------|-----------------|--------|------|
| 商業與管理群 | ~12,928 | 高 | **最大群，跨考最多** |
| 電機與電子群 | ~8,500 | 低 | 證照體系完整 |
| 機械群 | ~5,200 | 低 | 證照體系完整 |
| 餐旅群 | ~4,800 | 低 | 實務導向 |
| 設計群 | ~3,600 | 中高 | 競賽>證照 |
| 外語群 | ~2,400 | **最高** | **無證照，最大困境** |
| 資訊群 | ~3,200 | 低 | 證照+競賽都有 |
| 其他各群 | 各 <2,000 | 低-中 | — |

### 2.2 跨考（高職→學測）現象分析

**數據**：約 8% 的高職生會跨考學測，比例逐年上升。

**跨考集中在**：
- 商業與管理群（不知道商管也有好科大）
- 外語群（無技術士證照，覺得高職路走不通）
- 設計群（不知道設計競賽也能升學）

**跨考動因分析**：
1. **學校輔導不足**：高職學校偏重技術訓練，升學輔導資源少
2. **家長認知偏差**：「高職=只能做工」，不知道科大體系
3. **資訊不對稱**：不知道證照加分、技優甄審、競賽保送等管道
4. **同儕壓力**：看到同學跨考就跟著考

**商業機會**：如果能讓這 8% 的學生知道「高職路也能發光」，他們不需要跨考。這就是升學大師的價值。

### 2.3 外語群結構性劣勢

這是最值得關注的特殊案例：

- **無技術士證照**：台灣沒有「英文/日文技術士證照」
- **缺少競賽管道**：不像工科有技能競賽、不像設計有設計競賽
- **唯一加分路**：全民英檢/多益/日檢成績（但不算優待加分，只能放學習歷程）
- **結果**：外語群學生跨考率最高，因為覺得高職路走不通

**平台能做什麼**：
1. 幫外語群學生找到跨領域競賽（例如國際行銷、商務簡報）
2. 連結到可以加分的檢定考試（多益、日檢、全民英檢）
3. 引導到 C 類多元表現的善用（語言能力證明放入 C-5 或 D-3）
4. 發現外語+商管的跨域機會（外語群學生跨考商管類的科大反而有優勢）

## Part 3：產品定位 — 自我發現引擎

### 3.1 從 v3 到 v4 的進化

| 維度 | v3 | v4 |
|------|----|----|
| 定位 | 陪伴型升學平台 | 自我發現引擎 |
| 核心功能 | 記錄 → 被看見 → 被鼓勵 | 記錄 → 發現路徑 → 解鎖機會 |
| 價值主張 | 讓隱形人被看見 | 讓不知道路的人看見全部的路 |
| 差異化 | 情感陪伴 | 資訊解鎖 + 能力帳戶 |
| 新增功能 | — | 選考策略引擎、競賽資料庫、能力帳戶 |

### 3.2 能力帳戶（Ability Account）

**核心概念**：學生做的每一件事（證照、競賽、專題、實習、社團、志工）都是「存款」，存入能力帳戶。系統自動計算這些存款在升學中的「兌換價值」。

```
能力帳戶
├── 技術資產
│   ├── 證照（丙級 +5% / 乙級 +15% / 甲級 +25%）
│   ├── 技藝競賽獲獎（加分+門票）
│   └── 專業技能檢定
│
├── 學術資產
│   ├── 統測各科成績
│   ├── 課程成績
│   └── 跨考學測成績（如有）
│
├── 經歷資產
│   ├── 專題實作（B-1）
│   ├── 實習經驗
│   ├── 競賽參與（C-4）— 含外部競賽
│   ├── 社團/幹部（C-2/C-3）
│   └── 志工服務（C-6）
│
├── 軟實力標籤
│   ├── AI 自動標記：「主動學習者」「科系探索家」「實驗記錄者」
│   └── 來自經歷的推論，不是自評
│
└── 兌換價值
    ├── 即時試算：這些資產在目標校系的總分貢獻
    ├── 缺口分析：距離目標還差什麼
    └── 路徑推薦：哪條管道最適合你
```

### 3.3 自我發現引擎的使用者旅程

```
Day 1: 「你是誰？」
  → 選擇類群 + 年級
  → 系統顯示：「高職生有 6 種升學路，你知道嗎？」
  → 99% 的學生不知道 → 好奇心啟動

Week 1: 「你在做什麼？」
  → 記錄第一件事
  → 系統：「這件事在升學裡值 X 分」
  → 學生發現：原來我正在做的事有升學價值

Month 1: 「你有哪些路？」
  → 能力帳戶累積 3+ 件
  → 系統自動配對可行升學路徑
  → 學生：「原來我可以走技優甄審！」（最常見的驚喜）

Month 2-3: 「你的最優策略？」
  → 116 選考策略引擎上線
  → 根據能力帳戶推薦最佳考科組合
  → 缺口分析：你還需要什麼

Month 6+: 「你的專屬路線圖」
  → 以終為始的個人化計畫
  → 目標校系 → 回推需要準備什麼
  → 每週任務來自校曆和實際活動，不是 AI 憑空派任務
```

## Part 4：光域 UI 設計規範

### 4.1 設計哲學

靈感來源：Sky: Children of the Light（光遇）

**核心原則**：
- 溫暖、安全、不評價
- 探索是自由的，不是被指派的
- 成就是光點的收集，不是數字的累加
- 每個人的旅程都是獨特的

**不做的**：
- 不用壓力感的深色/科技風
- 不用排行榜/進度條/倒計時
- 不用紅色警告/未完成提示
- 不用遊戲化但帶壓力的元素（每日任務、連續登入）

### 4.2 色彩系統

```
主色調：暖光系
├── 主背景：#FFF8F0（晨光白）
├── 次背景：#FEF3E2（暖米色）
├── 卡片背景：#FFFFFF + 柔和陰影
├── 主文字：#3D3027（暖棕黑）
├── 次文字：#8B7D6B（暖灰棕）
├── 強調色：#E8A838（金光橙）— 用於成就/重要動作
├── 輔助色：#7EB8A0（苔綠）— 用於探索/路徑
├── 溫暖色：#D4836D（珊瑚）— 用於鼓勵/社交
└── 夜間模式：#1A1A2E → #16213E → #0F3460（星空漸層）
```

### 4.3 關鍵頁面設計

#### 探索地圖（首頁）

```
概念：光遇的世界地圖 → 升學路徑地圖
```

## 附錄 B：116 選考策略引擎演算法設計

### 問題定義

```
給定：
- 學生的目標校系列表 T = {(school_i, dept_i)}
- 可選考科集合 S = {國, 英, 數, 專一, 專二}
- 各目標校系的採計科目要求 R_i ⊆ S
- 學生的各科自評能力 W = {科目: 分數}

求解：
- 考科組合 C ⊆ S, |C| ∈ [2, 5]
- 最大化：覆蓋的目標校系數 + 加權後的預期總分
- 限制：C 必須滿足盡可能多的 R_i
```

### 演算法

```python
def recommend_exam_combination(targets, subjects, weights):
    """
    targets: [(school, dept, required_subjects, subject_weights)]
    subjects: available subjects
    weights: student's self-assessed ability per subject
    """
    best_combo = None
    best_score = -1

    # Enumerate all valid combinations (2-5 subjects)
    for size in range(2, 6):
        for combo in combinations(subjects, size):
            coverage = calculate_coverage(combo, targets)
            expected_score = calculate_expected_score(combo, weights, targets)
            total = coverage * 0.6 + expected_score * 0.4

            if total > best_score:
                best_score = total
                best_combo = combo

    return best_combo, best_score

def calculate_coverage(combo, targets):
    """How many targets can this combination qualify for"""
    covered = sum(1 for t in targets if set(t.required).issubset(set(combo)))
    return covered / len(targets)

def calculate_expected_score(combo, weights, targets):
    """Weighted expected score across covered targets"""
    total = 0
    count = 0
    for t in targets:
        if set(t.required).issubset(set(combo)):
            score = sum(weights.get(s, 50) * t.weights.get(s, 1.0) for s in combo if s in t.required)
            total += score
            count += 1
    return total / max(count, 1)
```
