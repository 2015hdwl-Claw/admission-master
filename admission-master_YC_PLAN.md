## Executive Summary

升學大師 v2.0 是一個**以終為始的高職升學路徑發現引擎**。

**核心產品突破**（2026-05-11 確立）：
- ❌ 舊模式：「輸入你現在有什麼 → 告訴你可以去哪裡」
- ✅ 新模式：「先說你想要去哪裡 → 告訴你現在差什麼」

**我們不會告訴你你辦不到。我們只會告訴你每一條路需要付出什麼代價。然後你自己選擇。**

**與其他升學網站的根本差異**：
- 其他網站：根據你現在的成績，你可以上這些學校。接受現實吧。
- 升學大師：不管你現在在哪裡，6 種不同的路都可以抵達你想要的目標。

## Phase 1: 構思與確認 ✅

### 1.1 產品定位

**最終定位**: 以終為始的升學路徑發現引擎

**核心問題**: 99% 高職生不知道自己有 6 種升學管道

**解決方案**: 
1. 學生先選擇目標科系
2. 系統展示所有 6 種入學管道
3. 盤點學生現有條件
4. 計算差距並生成行動計畫

### 1.2 目標用戶

**主要用戶**: 台灣高職生（免費使用）
**付費用戶**: 家長（成長報告、AI 輔導）

### 1.3 核心功能

1. **免費發現之旅**（6 步驟流程，光域風格）
2. **科系入學要求資料庫**（四技二專所有科系）
3. **差距分析引擎**（永遠不說「你不行」）
4. **個人化行動計畫**（倒數計時時程表）

### 1.4 功能融合策略

**保留並重新定位的頁面**:
- `/first-discovery` → **以終為始的發現之旅**（核心產品）
- `/explore` → **職群探索**
- `/portfolio` → **路徑準備材料**
- `/ability-account` → **Planner 能力中心**
- `/roadmap` → **升學路徑時間線**
- `/interview` → **個人申請準備**
- `/pathways` → **6 種管道總覽**
- `/department/[slug]` → **職群科系介紹**
- `/quiz` → **職群發現測驗**

**移除的頁面**: `/analyze`, `/business-strategy`, `/calendar`, `/guide`, `/results`, `/strategy`, `/teacher`, `/test`, `/demo`

**暫時隱藏**: `/parent`, `/parent/register`, `/pricing`

## Phase 2: 規劃與設計 ✅

### 2.1 核心產品：免費發現之旅

#### 六步驟流程（光域風格）

| 步驟 | 名稱 | 使用者行為 | 背景色 |
|------|------|-----------|--------|
| 1 | 我想去哪裡？ | 搜尋並選擇 1-3 個目標科系 | 淡藍漸層 |
| 2 | 這個目標需要什麼？ | 展開 6 種入學管道和門檻 | 淡紫漸層 |
| 3 | 我現在有什麼？ | 盤點年級、成績、證照、競賽、專題 | 淡綠漸層 |
| 4 | 我的差距在哪裡？ | ✨ 魔法時刻：差距分析 + 機率預測 | 淡金漸層 |
| 5 | 我需要完成什麼？ | 個人化待辦清單 | 淡橙漸層 |
| 6 | 我的時間表是？ | 倒數計時行動計畫 | 淡粉漸層 |

#### 光域風格 UI 設計原則

1. **一次一個問題**：全螢幕，沒有導覽列，一次只顯示一個輸入項，90% 內容 / 10% 介面
2. **沒有按鈕的介面**：完成輸入自動前進，滑動導航，永遠看不到「送出」「下一頁」
3. **流暢的狀態過渡**：淡入+滑入動畫、進度條平滑增長、背景色隨步驟漸變
4. **隨時可以回頭**：任何時候滑回去修改，修改後自動更新所有下游計算
5. **沒有評斷**：永遠只說「如果你想要這個，你需要做什麼」，差距 = 機會

### 2.2 科系入學要求資料庫

**資料結構**：
- 大學科系基本資訊（校名、科系名、科系代碼）
- 6 種入學管道各自的門檻和條件
- 歷史數據（去年最低分、平均分、錄取率、名額、報名人數）
- 申請時程（開始日期、截止日期、考試日期、放榜日期）
- 隱藏規則（加權方式、特殊偏好）

**目前狀態**：MVP 使用 mock data，35 個科系涵蓋 10 所大學、15 個職群

### 2.3 差距分析演算法

1. **絕對門檻過濾器**：先移除完全不符合資格的管道
2. **適合度評分器**：計算每個管道的相對優勢
3. **差距分析器**：精確計算每個條件的缺口和可行性
4. **機率預測器**：基於歷史數據的錄取機率估計

**核心原則**：永遠不說「你不行」，只說「如果你完成這些，你可以到達」

### 2.4 3 角色系統設計

#### Explorer (探索者) - 發現階段
- 目標：「原來我有這些選擇？」
- 核心體驗：免費發現之旅、職群探索、6 種管道教育
- 核心頁面：`/first-discovery`, `/explore`, `/pathways`, `/quiz`

#### Planner (規劃者) - 規劃階段
- 目標：「如何最大化我的升學機會？」
- 核心體驗：能力中心、材料準備、時間線規劃
- 核心頁面：`/ability-account`, `/portfolio`, `/roadmap`, `/interview`

#### Guide (引路人) - 引導階段
- 目標：「幫助學弟學妹避免走彎路」
- 核心體驗：分享經驗、病毒傳播（待開發）

### 2.5 技術架構

**前端**: Next.js 14 App Router + TypeScript + Tailwind CSS  
**後端**: Supabase (PostgreSQL + Auth)  
**部署**: Vercel (前端) + Supabase Cloud (後端)  
**生產環境**: https://admission-master-ecru.vercel.app/
