## Phase 1: 構思與確認 ✅
*目標: 驗證想法可行性，評估市場機會*

### 核心決策
- ✅ **產品重新定位**: 從「學習歷程管理平台」→「高職升學路徑發現引擎」
- ✅ **核心問題**: 99% 高職生不知道自己有 6 種升學管道
- ✅ **目標用戶**: 高職生（免費）+ 家長（付費）
- ✅ **角色系統**: Explorer → Planner → Guide (簡化為 3 角色)

### 市場分析
- ✅ **痛點驗證**: 高職升學資訊不對稱，多數學生只知道聯考和推甄
- ✅ **解決方案**: 6 種升學管道教育 + 個人化路徑發現
- ✅ **差異化**: 從「成績管理」轉向「路徑發現」思維

### 檢核項目
- ✅ **產品定位明確**: 升學路徑發現引擎
- ✅ **目標用戶清晰**: 高職生為主，家長為付費對象
- ✅ **核心問題識別**: 99% 高職生不知道的升學管道
- ✅ **解決方案可行**: 技術架構已驗證 (Next.js + Supabase)

## Phase 2: 規劃與設計 ✅
*目標: 詳細產品規格，技術架構設計*

### 3角色系統設計
- ✅ **Explorer (探索者)**: 發現階段 - 「原來我有這些選擇？」
- ✅ **Planner (規劃者)**: 規劃階段 - 「如何最大化我的升學機會？」
- ✅ **Guide (引路人)**: 引導階段 - 「幫助學弟學妹避免走彎路」

### 漸進式實施策略
- ✅ **Phase 1** (Week 1-2): Explorer Experience - 首页改造 + 第一次發現頁面
- ✅ **Phase 2** (Week 3-4): Planner Functionality - 規劃中心 + 能力帳戶重新設計
- ⏳ **Phase 3** (Week 5-6): Guide System - 引導平台 + 故事系統

### 技術架構
- ✅ **資料庫**: Supabase PostgreSQL (已有角色系統 schema)
- ✅ **前端**: Next.js 14 App Router + Tailwind CSS
- ✅ **認證**: Supabase Auth (已實現職群選擇器整合)
- ✅ **部署**: Vercel (前端) + Supabase Cloud (後端)

### 檢核項目
- ✅ **系統架構設計**: 3角色漸進式實施
- ✅ **資料庫 schema 完成**: user_roles, role_evolution, xp_history
- ✅ **API 規格定義**: 職群映射、路徑計算邏輯
- ✅ **UI/UX 設計**: Explorer 主題、6 種管道展示
